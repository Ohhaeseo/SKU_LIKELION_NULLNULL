# 🚀 Seoul Nowcast AI — 제품 기획서 (PRD v2)

> **"AI로 헛걸음을 없앤다면?"**
> 서울을 여행하는 외국인의 '헛걸음'을 AI가 사전에 차단하는 실시간 도시 인텔리전스 플랫폼

---

## 1. 문제 정의 (Problem Statement)

### 1-1. 핵심 문제: "헛걸음"

외국인 관광객이 서울에서 겪는 가장 큰 좌절은 **"가보니까 사람이 너무 많아서 돌아온 경험"**, 즉 **헛걸음**이다.

| 헛걸음 유형 | 구체적 상황 | 빈도 |
|------------|-----------|------|
| **혼잡 헛걸음** | 홍대에 갔는데 인파에 치여 쇼핑 포기 | 매우 높음 |
| **날씨 헛걸음** | 한강공원 갔는데 비가 와서 되돌아옴 | 높음 |
| **정보 헛걸음** | 유명하다길래 갔는데 별로, 줄만 길게 섬 | 중간 |
| **교통 헛걸음** | 지하철 너무 붐벼서 3대를 보내고 겨우 탑승 | 중간 |

### 1-2. 왜 외국인인가?

한국인은 네이버 카페, 트위터, 에타 등 한국어 기반 실시간 커뮤니티를 통해 "지금 어디가 붐비는지"를 어느 정도 파악할 수 있다. 그러나 **외국인은 이 정보에 접근이 불가능**하다.

```
[정보 비대칭 Gap]

한국인:  네이버 실검 + SNS + 지인 정보 → "오늘 명동 행사라서 피하자"
외국인:  Google Maps만 의존 → "왜 이렇게 사람이 많지?" (도착 후 인지)
                                         ↑
                            이 갭을 AI가 메운다
```

### 1-3. 문제의 규모

- 2024년 기준 방한 외국인 관광객: **약 1,600만 명** (역대 최대)
- 서울 집중율: 약 80% → **서울 방문 외국인 약 1,280만 명/년**
- 이 중 "사람이 많아 일정 변경 경험" 추정: 약 60% (사용자 조사 기반 가설)

## 2. 솔루션 개요 (Solution Overview)

### 서비스명: **NoWhere** (Now + Here + No Wasted Trip)

> "Don't go nowhere — Go **NoWhere**."
> (아무 데나 가지 말고, 지금·여기에 맞는 최적의 장소로)

### 한 줄 정의

**30분 지연된 서울시 인구 데이터의 갭을, 실시간 지하철 승하차 + 날씨 데이터로 AI가 메워 "지금 이 순간"의 혼잡도를 예측하고, 외국인에게 최적의 장소·시간을 추천하는 모바일 앱.**

### Value Proposition Canvas

| 고객의 Pain | 우리의 Solution |
|------------|----------------|
| 가보니 너무 붐벼서 돌아옴 | 출발 전 실시간 혼잡도 확인 + AI 예측 |
| 비 오는 날 갈 곳을 모름 | 날씨 기반 실내/실외 장소 자동 추천 |
| 언제 가야 한가한지 모름 | AI 시간대별 혼잡 예측 ("16시에 가세요") |
| 현지 맛집·카페 정보 부족 | 로컬 커뮤니티 + AI 상권 추천 |
| 여행 동선 짜기 어려움 | AI가 혼잡도·날씨·거리 고려한 일정 자동 생성 |

## 4. 핵심 기능 (MVP Feature Set)

### 4-1. 🗺️ 실시간 혼잡도 지도 (Crowd Density Map)

지도 위에 서울 주요 50개 장소의 인구 밀집도를 **색상 원(Circle)**으로 실시간 표시.

| 색상 | 의미 | 인구 밀도 |
|------|------|----------|
| 🟢 초록 | 여유 | ~20,000명 |
| 🟡 노랑 | 보통 | 20,000~40,000명 |
| 🟠 주황 | 약간 붐빔 | 40,000~55,000명 |
| 🔴 빨강 | 붐빔/매우 붐빔 | 55,000명~ |

**데이터 출처**: 서울시 실시간 도시데이터 API (`LIVE_PPLTN_STTS`)

### 4-2. 🤖 AI 나우캐스트 (Nowcasting Engine)

서울시 데이터는 **30분 지연**된다. 이 갭을 AI가 메운다.

```
┌─────────────────────────────────────────────────────┐
│                AI Nowcasting Pipeline                 │
│                                                       │
│  [서울시 인구 T-30]  ──┐                              │
│                        ├──→  시계열 AI  ──→  현재 추정값 │
│  [지하철 승하차 T-0]  ──┤     (LSTM)        (Nowcast)  │
│                        │                              │
│  [날씨 데이터 T-0]   ──┘                              │
│                                                       │
│  Output:                                              │
│  ① 현재 추정 인구 (Now)                               │
│  ② 30분 후 예측 (Forecast)                            │
│  ③ 혼잡도 레벨 (여유/보통/붐빔/매우붐빔)               │
└─────────────────────────────────────────────────────┘
```

**왜 지하철 데이터가 핵심 Proxy인가?**
- 서울 핫플레이스 방문자의 약 70%가 지하철로 이동
- 홍대입구역 하차 인원이 급증 → 15분 뒤 홍대 거리 인구 급증
- 이 "선행 지표"를 AI가 학습하여 30분 갭을 실시간으로 보정

**왜 날씨가 중요한가?**
- 비가 오면: 야외 장소 인구↓, 지하상가·쇼핑몰↑
- 폭염이면: 한강공원↓, 에어컨 있는 실내↑
- 날씨는 인구 흐름의 **방향 전환 변수**

### 4-3. 💡 AI 스마트 추천 (Smart Recommendation)

단순히 "여기 붐빈다"만 알려주는 것이 아니라, **대안까지 제시**한다.

```
🔴 홍대입구역 — 현재 매우 붐빔 (6.2만명)
   30분 후에도 혼잡 유지 예측 (주말 피크)

   💡 AI 추천:
   "성수동 카페거리로 가세요! 지금 여유 (1.2만명)
    홍대에서 2호선 4정거장, 15분 소요.
    인기 카페 '대림창고' 평점 4.7 ⭐"
    
   ⏰ 타임 넛지:
   "홍대는 오늘 18시 이후 여유 예측.
    지금 성수 → 18시 홍대 동선을 추천합니다."
```

### 4-4. 🌤️ 날씨 기반 스마트 필터

```
🌧️ 오후 3시 비 예보 감지!

   ❌ 야외 추천 해제: 한강공원, 북촌한옥마을
   ✅ 실내 추천 활성: 코엑스몰, 스타필드, 국립박물관

   "비 오기 전 2시까지 야외 일정을 마치고,
    3시부터는 코엑스몰(지금 여유)을 추천합니다"
```

## 5. 서브 기능 (Phase 2)

### 5-1. 🏘️ 동네 커뮤니티 (Local Community)

- 로컬 주민이 직접 올리는 "숨은 맛집", "지금 여기 한가해요" 정보
- 외국인에게는 영어로 자동 번역되어 표시
- **상권 광고 모델**: 로컬 카페·음식점이 "외국인 10% 할인" 같은 프로모션 게재

### 5-2. 📅 AI 일정 자동 생성

- 입력: "3박 4일, 서울, 한식·카페·쇼핑 좋아함"
- AI가 혼잡도 예측 + 날씨 + 교통 동선을 고려하여 최적 일정 생성
- "월요일 오전: 경복궁 (여유 예측) → 점심: 삼청동 맛집 → 오후: 성수 카페 (화요일은 붐빔)"
