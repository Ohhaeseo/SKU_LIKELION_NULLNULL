# 덜붐 서비스 기획서

> 서울을 덜 붐비게 이동하는 방법  
> 외국인 관광객을 1차 타겟으로 하는 서울 혼잡 회피형 AI 여행 비서

---

## 1. 한 줄 요약

**덜붐**은 서울의 실시간 혼잡도, 날씨, 대중교통 정보를 바탕으로 사용자가 “지금 가도 괜찮은 곳”과 “피해야 할 곳”을 빠르게 판단하도록 돕는 AI 여행 비서 서비스다.

핵심 AI 비서 이름은 **길라미 AI**다.

---

## 2. 기획 배경

서울은 관광지, 핫플레이스, 공연장, 팝업스토어, 쇼핑 거리, 한강공원처럼 사람들이 몰리는 장소가 많다.

하지만 사용자가 실제로 이동하기 전에는 다음 정보를 한 번에 알기 어렵다.

- 지금 그 장소가 얼마나 붐비는지
- 30분 뒤 더 붐빌 가능성이 있는지
- 비가 오거나 날씨가 나빠질 때 어디로 바꿔야 하는지
- 외국인 관광객이 이용하기 쉬운 경로인지
- 조금 돌아가더라도 더 안전하고 쾌적한 길이 있는지

기존 지도 앱은 빠른 길과 장소 검색에는 강하지만, **혼잡 회피, 안전성, 대체 장소 추천**까지 한 번에 판단해주지는 못한다.

---

## 3. 사용자 조사 설계

### 조사 목적

사용자 조사를 통해 서울 이동과 관광 중 발생하는 실제 pain point를 파악한다.

특히 주 타겟인 **외국인 관광객**이 서울에서 느끼는 불편함을 중심으로 검증한다.

### 핵심 조사 질문

- 서울을 다니면서 사람이 많아서 가려던 곳을 못 가거나 계획에 차질이 생긴 적이 있는가?
- 홍대, 성수 등 서울에서 이동할 때 주로 어떤 대중교통을 이용하는가?
- 서울 핫플레이스를 방문할 때 “지금 이 장소에 사람이 얼마나 있는지” 미리 알고 싶었던 적이 있는가?
- 날씨가 안 좋을 때 다른 장소를 찾아야 하는 상황에서 어려움이 있었는가?
- 추천 장소가 원래 계획한 곳보다 쾌적하고 안전하다면 따를 의향이 있는가?
- 이런 앱이 실제로 생기면 사용할 계획이 있는가?

### 검증해야 할 가설

- 외국인 관광객은 장소 정보보다 “지금 가도 되는지”를 더 알고 싶어 한다.
- 사용자는 최단 경로보다 덜 붐비고 안전한 경로를 선택할 의향이 있다.
- AI 추천을 따르려면 단순 추천보다 이유, 근거, 대체 행동이 함께 필요하다.
- 날씨 악화, 행사 종료, 출퇴근 시간에는 대체 장소 추천 니즈가 강해진다.

---

## 4. 핵심 Pain Points

### Pain Point 1. 혼잡 정보가 흩어져 있다

- 실시간 혼잡도, 행사 정보, 교통 정보, 날씨 정보가 각각 다른 채널에 있다.
- 사용자가 직접 여러 앱을 오가며 판단해야 한다.
- 관광객은 어떤 정보가 신뢰할 만한지 판단하기 어렵다.

### Pain Point 2. 외국인 관광객은 서울의 혼잡 맥락을 알기 어렵다

- “홍대입구 9번 출구”, “잠실역 행사 종료 시간”, “명동 주말 저녁” 같은 로컬 맥락을 알기 어렵다.
- 한국어 안내와 지하철 출구 체계가 낯설다.
- 붐비는 상황이 단순 불편인지, 피해야 할 위험인지 구분하기 어렵다.

### Pain Point 3. 최단 경로가 항상 좋은 경로는 아니다

- 빠르지만 환승역이 붐비거나, 골목이 좁거나, 행사 인파와 겹칠 수 있다.
- 기존 지도 앱은 이동 경험의 쾌적함을 충분히 설명하지 못한다.
- 사용자는 “몇 분 더 걸리더라도 덜 붐비는 길”을 알고 싶어 한다.

### Pain Point 4. 날씨가 나빠지면 대체 장소를 찾기 어렵다

- 비, 폭염, 한파 상황에서 실내 장소를 다시 찾고 비교하는 데 시간이 오래 걸린다.
- 관광객은 가까운 대체 장소의 분위기, 거리, 영업 여부를 빠르게 판단하기 어렵다.

### Pain Point 5. AI 추천을 신뢰하기 위한 설명이 부족하다

- “혼잡도 높음”만으로는 행동으로 이어지기 어렵다.
- 사용자는 왜 피해야 하는지, 얼마나 더 걸리는지, 어디로 가면 좋은지 알고 싶어 한다.

---

## 5. 문제 정의

서울 시민과 방문객은 이동과 관광 계획을 세울 때 시간뿐 아니라 **혼잡도, 안전성, 날씨, 대기시간, 언어 접근성**을 함께 고려해야 한다.

하지만 기존 지도와 관광 앱은 이 정보를 통합해 “지금 이동해도 되는지”와 “어디로 바꾸면 좋은지”까지 행동 중심으로 안내하지 못한다.

따라서 사용자는 실시간 인파, 대중교통 혼잡, 행사 정보, 날씨, 보행 환경을 종합해 **지금 가장 쾌적하고 안전한 선택지**를 알려주는 서비스가 필요하다.

## 6. 솔루션 개요

**덜붐**은 서울의 혼잡 데이터를 기반으로 사용자가 지금 가도 괜찮은 장소와 경로를 판단하도록 돕는 AI 여행 비서 서비스다.

### 핵심 가치

- 어디가 붐비는지 보여준다.
- 언제 가면 덜 붐비는지 예측한다.
- 붐비면 어디로 바꾸면 좋은지 추천한다.
- 외국인 관광객이 바로 이해할 수 있는 행동 안내를 제공한다.

### 핵심 사용 흐름

1. 사용자가 홈에서 현재 붐비는 장소와 현재 위치 기반 날씨를 확인한다.
2. 지도 탭에서 서울 주요 지역 밀집도를 확인한다.
3. 길찾기 탭에서 현재 위치와 목적지를 선택한다.
4. 대중교통 이용방법을 확인한다.
5. AI비서 탭에서 목적지 분석, 경고 메시지, 대체 장소 추천을 확인한다.

## 7. 핵심 기능

### 7.1 홈

홈은 다른 화면보다 더 크게 보이는 중심 화면이다.

제공 정보:

- 현재 붐비는 장소
- 현재 위치 기반 날씨
- 혼잡 지역 요약
- 대체하기 좋은 장소
- 지도와 길찾기로 이동하는 빠른 버튼

### 7.2 지도

서울시 주요 지역의 밀집도를 샘플로 보여준다.

초기 MVP에서는 실제 지도 SDK 없이도 밀집도를 이해할 수 있는 시각화로 시작한다.

표시 지역 예시:

- 홍대
- 성수
- 명동
- 강남
- 이태원
- 경복궁
- 여의도 한강공원

### 7.3 길찾기

네이버 지도처럼 현재 위치와 목적지를 선택할 수 있다.

선택 완료 후 제공 정보:

- 대중교통 이용방법
- 빠른 경로
- 균형 추천 경로
- 덜 붐비는 경로
- 쾌적 점수
- 목적지 분석 결과로 이동하는 버튼

### 7.4 AI비서

AI비서 탭에서는 목적지에 대한 분석 결과를 보여준다.

경고 메시지가 있는 경우:

- 혼잡 경고
- 예상 위험 시간
- 경고 이유
- 대체 행동
- 대체 장소 추천

경고 메시지가 없는 경우:

- 추천 맛집
- 추천 장소
- AI가 예측한 방문 팁
- 날씨 기반 안내 문구

### 7.5 마이페이지

기본 앱에 필요한 메뉴를 포함한다.

- 고객센터
- 설정
- 알림 설정
- 저장한 장소
- 개인정보 및 안전
- 데이터 연결 상태
- 언어 설정
- 위치 권한 설정

## 8. 타겟 사용자

### 전체 타겟

서울시에 거주하거나 서울에 놀러오는 모든 인원.

### 주 타겟

서울의 지리, 언어, 혼잡 시간대에 익숙하지 않은 외국인 관광객.

- 주말 나들이객
- 아이 또는 노약자 동반 가족
- 공연, 축제, 팝업스토어 방문자

## 10. 유사 서비스 대비 차별화

### 기존 서비스

| 서비스 | 강점 | 한계 |
|---|---|---|
| 네이버지도 / 카카오맵 | 장소 검색, 대중교통, 길찾기 | 혼잡 회피와 대체 일정 추천은 제한적 |
| 구글맵 / 트립어드바이저 | 외국인 관광객의 기본 탐색 도구 | 서울 로컬 혼잡 맥락 반영이 약함 |
| 서울 실시간 도시데이터 | 공공데이터 기반 혼잡 정보 | 일반 관광객이 행동으로 옮기기 어렵다 |
| 관광 공식 앱 / 웹 | 관광지와 축제 정보 제공 | 실시간 혼잡 대응이 약함 |

### 덜붐의 차별화

1. **혼잡 회피 중심**
   - 빠른 길이 아니라 덜 붐비고 안전한 길을 핵심 가치로 둔다.

2. **행동 권고형 AI**
   - “혼잡도 높음”에서 끝나지 않고 “10분 대기”, “다른 출구 이용”, “대체 장소 이동”을 제안한다.

3. **외국인 관광객 특화**
   - 서울의 로컬 혼잡 맥락을 쉬운 언어로 설명한다.
   - 다국어 안전 안내와 여행 상황별 추천을 제공한다.

4. **대체 일정 추천**
   - 목적지가 붐비면 비슷한 분위기의 덜 붐비는 장소를 바로 추천한다.

5. **서울 특화 데이터 맥락**
   - 축제, 공연장, 대학가, 한강, 관광지, 출퇴근 패턴 등 서울의 반복 혼잡 패턴을 반영한다.

## 11. AI 비서 네이밍

### 최종 추천: 길라미 AI

의미:

- “길” + “알림이”
- 한국어 사용자에게 직관적이다.
- 경고 기능이지만 딱딱하지 않고 친근하다.
- 외국인용으로는 **Gilami AI**로 표기할 수 있다.

예시 문구:

```text
길라미 AI가 현재 홍대입구역 9번 출구 혼잡을 감지했어요.
성수 카페거리로 일정을 바꾸면 이동 시간이 12분 늘지만 혼잡도는 낮아집니다.
```

## 12. 앱 서비스 이름 후보

### 1순위: 덜붐

이유:

- 서비스 가치가 바로 이해된다.
- 짧고 기억하기 쉽다.
- “덜 붐비는 서울 이동”이라는 핵심 메시지와 직접 연결된다.
- 한국 시장 MVP에 적합하다.

슬로건:

```text
서울을 덜 붐비게 이동하는 방법.
```

### 추가 후보

| 이름 | 장점 | 우려 |
|---|---|---|
| 여유길 | 안전하고 편한 이동 이미지 | 다소 일반적 |
| CrowdWay Seoul | 외국인 관광객에게 의미 전달 쉬움 | 한국적 개성 약함 |
| 서울숨길 | 숨 쉴 수 있는 길 이미지 | 설명이 조금 필요 |
| 한산길 | 목적이 명확함 | 공공서비스처럼 느껴질 수 있음 |
| 루트숨 | 감성적이고 기억 가능 | 의미 전달에 보조 카피 필요 |

## 14. MVP 범위

## 15. 성공 지표

| 지표 | 의미 |
|---|---|
| AI 추천 수용률 | 사용자가 AI가 제안한 경로/장소를 선택한 비율 |
| 혼잡 경고 클릭률 | 경고 메시지가 실제 행동으로 이어지는 정도 |
| 대체 장소 추천 클릭률 | 사용자가 대체 장소를 탐색한 비율 |
| 목적지 변경률 | 혼잡 회피 추천 후 목적지를 바꾼 비율 |
| 앱 재사용률 | 여행 중 반복 사용 가능성 |
| 외국인 관광객 만족도 | 주 타겟의 체감 가치 |
| 계획 차질 감소율 | 사용자가 느낀 일정 실패 감소 정도 |

## 16. 남은 검증 과제

- 외국인 관광객이 AI 추천을 따르는 조건은 무엇인가?
- 위치 공유에 대한 거부감은 어느 정도인가?
- 혼잡 회피를 위해 감수 가능한 추가 이동 시간은 몇 분인가?
- 국가별로 교통, 언어, 결제, 안전 니즈가 어떻게 다른가?
- 사용자는 로컬 추천을 원하는가, 안전하고 검증된 관광객 친화 장소를 원하는가?
- “혼잡 경고”가 불안을 주지 않으면서 행동을 유도하려면 어떤 문구가 적절한가?

## 17. 현재 앱 네비게이션 구조

하단 네비게이션 바 순서:

```text
지도 → 길찾기 → 홈 → AI비서 → 마이페이지
```

### 화면별 역할

| 화면 | 역할 |
|---|---|
| 지도 | 서울시 밀집도 샘플 표시 |
| 길찾기 | 현재 위치와 목적지 선택, 대중교통 경로 표시 |
| 홈 | 현재 붐비는 장소와 현재 위치 기반 날씨 표시 |
| AI비서 | 목적지 분석, 경고 메시지, 추천 문구 표시 |
| 마이페이지 | 고객센터, 설정, 알림 설정 등 기본 기능 |

## 18. 최종 포지셔닝

**덜붐은 서울 관광객을 위한 실시간 혼잡 회피형 AI 여행 동반자다.**

지도 앱과 직접 경쟁하기보다, 지도 위에 얹히는 **안전, 혼잡, 대체 코스 특화 레이어**로 차별화한다.
